# A 股小资金短线交易系统 —— Project Plan v1

> 本文是在《交易系统_施工总结_v1.md》（架构总结）基础上，经过一轮"技术选型 + 系统形态 + 策略方法论"的细化讨论后产出的**施工计划**。施工总结回答"做什么、为什么做"，本文回答"怎么落地"。
>
> 一句话定位不变：**这不是自动交易机器人，而是辅助"我"做决策的系统——约束纪律、放大信息、解释概念；最终扳机永远由我自己扣。**

## 1. 系统形态与技术栈（已锁定）

| 维度 | 决定 | 理由 |
|---|---|---|
| 决策模式 | 系统辅助、人拍板，不全自动 | 要练的是我自己 |
| 客户端 | 原生 **SwiftUI iOS App**，开发者账号直装自有设备 | 全设备苹果 + 有开发者账号；要"自己 App 的系统通知" |
| 推送 | ECS **直连 APNs** → App 系统通知 | 自有壳；Bark 仅作临时验证/备份 |
| 后端宿主 | **阿里云 ECS（1C2G / 30G SSD）** | 已有；轮询监控 + 调 LLM API 绰绰有余 |
| 进程形态 | systemd 守护进程 + FastAPI 后端 + 交易时段调度 | 常驻、免运维尾巴尽量小 |
| 轮询频率 | 交易时段**每 1 分钟** | 容忍延迟 1–3 分钟，大阴线直落极少见 |
| 数据存储 | **SQLite**（单文件，零运维） | 单人、小量级 |
| 运行时模型 | **DeepSeek**（前置词 + skills 武装成 A 股短线专家） | 国内 ECS 直连、便宜、无墙；中文语境足够 |
| 下单/持仓 | **不接券商 API**（A 股无开放 API），手动录入 | 与"扳机在我"一致 |

## 2. 总体架构

三块：①你的苹果设备　②阿里云 ECS（系统大脑）　③外部服务。

```
[苹果设备]                 [阿里云 ECS · 系统大脑]              [外部服务]
 系统通知(APNs接收) <──硬线推送── 监控守护进程(每1min·只判硬线) ──拉价── 新浪/腾讯(实时价·免费)
 原生App(3卡片+对话) <──看板/对话──> 后端API(编排·服务App)    <────> DeepSeek(中间地带·复盘)
 快捷指令(持仓录入) ──录入──────────> 后端API ───────────────── Tushare付费(历史·资金·板块)
                                      SQLite(持仓·流水·复盘·记忆)
                                      周复盘任务(独立上下文)
                              [券商到价提醒：独立兜底，不接入系统]
```

- **闭环路径**：周复盘任务 → 写入 SQLite → 后端 API 读出 → 注入下周交易上下文。
- **券商到价提醒**：刻意独立于系统之外，自制监控万一挂了仍有它兜底；它是安全网，不是组件。

## 3. 脊椎：报警服务（机械层）

**职责**：交易时段每 1 分钟拉取在持仓票的实时价，命中硬线即推送，绝不做软判断。

**触发条件（硬线，机械执行）**：

| 触发 | 动作 |
|---|---|
| 单票 -5% | 止损提醒（极强趋势上限放宽至 -10% 封顶） |
| 单票 +15% | 止盈提醒（必走） |
| **持仓满 3 交易日 → 第 4 日** | **清仓提醒（无条件）** |

**配套**：
- 数据源：免费新浪/腾讯实时行情；多源/降级兜底以防限频或接口失效。
- **每日收盘后"心跳"推送**（报平安）：监控哪天挂了当天即可发现。
- **券商 App 到价提醒**：人工设置，独立兜底（安全考量，止损是铁律，不押单一自制服务）。
- 通知哲学：**硬线主动推；中间地带状态你主动拉**（上班不被中间地带骚扰）。

## 4. 持仓录入

- 入口：**苹果快捷指令 / Siri**（买完一句话录入），App 内做查看/管理。
- 录入字段：`代码 / 买入价 / 数量(可选) / 止损线 / 进场理由(一句话)`。
- **系统自动快照**：录入瞬间抓取并存下当时的**形态 + 资金状态**——供 Reviewer 事后评判"进场质量/是否凭感觉"。

## 5. 选股系统（两段式流水线）

> 设计自洽点：形态/资金/消息基本交 LLM 判断，但 LLM 扫不了全市场 4000+ 只。故先**宽松机械粗筛**缩名单，再 **LLM 精判**。

### ① 机械粗筛（宽松，只负责"别漏"）
- 形态近似条件：放量突破 / 创 N 日新高 / 站上关键均线 等好计算的项（用**宽区间**，不卡死数）。
- 资金宽条件：近几日主力净流入为正。
- **硬排除**：
  - 板块/类型黑名单：**300、688、白酒、ST**。
  - **高位排除**：前期累计涨幅 **≥100% → 完全不做**；**≥50% → 警告（降级）**。**只做低位平台。**
  - 资金硬规则：**当日小单净流入特别多 → 直接排除**（散户接盘 = 疑似出货诱多）。
- 产出：几十只候选。

### ② LLM 精判（DeepSeek，挑准）
顺序：**形态质量 → 资金确认 → 消息面排雷 → 出建议**。

---

## 6. 三维度细则（DeepSeek 的"专业弹药" / skills 内容）

### 6.1 形态面（主轴）
- 主轴：**平台突破 / 底部放量启动**；**剔除左侧抄底**。
- **进场时机：A/C/D 有机结合，按"看盘时机"自适应**：
  - A 撞上盘中突破 → 顺势进；
  - C 尾盘看到放量站稳平台之上 → 进；
  - D 昨日没进、次日形势仍强 → 跟进。
  - **B（等回踩确认）排除**：强势股不回踩，会踏空。
- **平台定义 / 突破有效性：不设死阈值**，LLM 具体情况具体分析（横盘时长、振幅、放量倍数、站稳与否均给宽区间辅助，不硬卡）。
- 止损位：以 **-5%** 为准；若破平台但整体仅亏 2–3%，落入中间地带由 LLM 判。

### 6.2 资金面（确认器）
- **只看两个信号：主力净流入 + 换手率**（不看北向、不看龙虎榜）。
- 时间窗：**连续几日净流入**（持续性优先），但**当日不能大幅净流出**。
- 顺序：**先 K 线（形态）→ 再资金确认**。
- 硬规则：当日小单净流入特别多 → 排除（已在粗筛）。

### 6.3 消息面（排雷，非入口）
- **角色降权**：资金 + 技术是主轴；消息/板块只做**最后排雷**。
- 题材热度：看**板块涨幅 + 板块资金流入**（结构化、轻量，不碰新闻 NLP）。
- **一日游判断**：交 LLM 的判断力，并允许它**读股吧/博主舆情**感知情绪（best-effort，仅对候选少量做）——这是"信息放大器"职能落地。
- 个股消息：**不作买入理由，只做排雷**（例：被监管警告 → 直接不进）。
- "泡沫明显"：LLM 主观判（半导体等题材股普遍高 PE，不做一刀切估值过滤）。

---

## 7. LLM 决策层（DeepSeek）

- **前置词 + skills**：把第 6 节方法论写成系统提示 + 技能模块，武装成 A 股短线专家。
- **中间地带（-5%~+15%）辅助 = B 剂量**：给信息 + 明确建议（拿/减/走）+ 写清理由，扳机仍由我扣。
  - 最看重：**量能是否萎缩 + 主力资金还在不在**（最优先，非唯一）；兼看是否仍在买入逻辑里、大盘环境、距两条线远近。
- **利润回撤保护**：不设自动规则；冲高回落时**我主动问**，LLM **看分时线趋势**帮我分析（分时数据免费按需拉）。
- **反情绪教练职责（启用）**：发现我想破纪律硬扛（"总觉得会反弹"）时，**主动调出复盘历史 + 铁律怼我**（如"你上次就是这么想的，亏了 4000"）。与闭环（复盘结论注入）联动。

---

## 8. 离场规则（铁律 + 中间地带）

- 止损：单票 **-5% 必走**；极强趋势上限放宽 **-10% 封顶**。
- 止盈：**+15% 必走**。
- **时间止损：满 3 交易日，第 4 日无条件清仓。**
- 中间地带（-5% ~ +15%）：核心战场，由 LLM 辅助（见第 7 节）。
- 因 T+1，离场动作集中在次日；不机械化"高开/低开"。

---

## 9. Reviewer 周复盘模块

- **隔离**：独立 Reviewer Agent（DeepSeek，独立上下文），**只吃客观交易流水 + 铁律，绝不读盘中聊天**；周末定时触发；周末前提醒到复盘时间。
- **双维度评判**：
  - **纪律（过程）**：决定行为对错 → 守纪律=肯定/不标红，破纪律=标红。
  - **盈亏（结果）**：决定选股灵不灵 → 赚钱总结经验，亏钱想教训。
- **打分**：量化分数 + 标红标签（两者都要）。
  - 维度：①进场是否合规（形态/资金/非高位/非黑名单）②离场是否守纪律（-5%/+15%/第4日清）③中间地带有无情绪化硬扛 ④整体是否破铁律 ⑤**结果（盈亏/落袋质量）计入**。
  - 标尺：**100 分 = 守纪律 + 吃满 +15% 落袋 + 走得坚决**；**-5% 止损即便纪律满分也仅 80–90 分**（进场/选股质量有提升空间）；具体分数 LLM 定。
- **标红清单**：该走（-5%）没走 / 超 3 日没清 / 追了高位（进 ≥50% 警告区）/ 进场不符合形态资金 / 没记理由凭感觉乱进。
- **肯定清单**：亏钱但严格止损 / 守住第 4 日清仓 / 该走就走哪怕错过后续反弹。
- **产出**：每周一份**小报告** = 每笔点评 + **本周纪律执行率（带趋势，看是否进步）** + 盈亏总结（经验/教训）+ 一条"下周注意"。
- **闭环（A）**：复盘结论自动写入 SQLite，注入下周交易上下文。

## 10. 数据层

- **实时价（脊椎）**：免费新浪/腾讯；多源降级兜底。
- **分时/分钟线**：免费，按需（利润回撤分析时拉）。
- **Tushare 付费**（按需要的接口倒推套餐，见 TODO）：
  - `moneyflow`：主力/大单/小单净额（资金面）。
  - `daily_basic`：换手率、涨跌幅、PE/PB 等。
  - 板块涨幅 + 板块资金流入（消息面排雷）。
  - `trade_cal`：交易日历（驱动交易时段调度 + 时间止损天数）。
  - 日线（形态）；概念成分（可选）。
- **舆情**：股吧/雪球/博主，精判阶段对候选少量抓取（best-effort）。

---

## 11. 存储（SQLite 表草案）

- `positions`：当前持仓（代码、买入价、数量、止损线、进场理由、进场形态/资金快照、买入日、持仓交易日计数）。
- `trades`：历史交易流水（开/平仓价、时间、是否守各条线、盈亏、是否破纪律）。
- `reviews`：每周复盘结论（评分、标红项、纪律执行率、经验/教训、下周注意）。
- `memory`：注入交易上下文的闭环结论 / 长期记忆。

## 13. 已决策的"砍掉/不做"项

- **全自动回测（策略验证回测）：砍。** 决策是"系统辅助、人拍板"，机器全自动跑规则对这套系统无意义。
- **不接券商下单/持仓 API**（A 股无开放 API）。
- 历史重放沙盒：推到 V2，定位陪练不是裁判（样本太少，得不出统计期望）。
- 若将来仍做任何回测，务必规避三坑：**涨停买入假设、未来函数、交易成本**。

## 15. 主要风险

- **自制监控悄悄挂** → 每日心跳 + 券商到价提醒双兜底。
- **免费实时接口不稳/限频** → 多源备份 + 降级。
- **DeepSeek 调用**：国内 ECS 直连无障碍；若将来改用 Claude 需自建中转。
- **纪律迁移风险**（V2 沙盒）：知道是历史 + 无真金白银压力，纪律未必迁移到实盘；选段有幸存者偏差。

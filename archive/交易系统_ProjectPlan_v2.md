# A 股小资金短线交易系统 —— Project Plan v2

> 在 v1 基础上,经过一次"用户视角跑完整交易周"的结构走查后修订。走查照出了三处真缺口(全买全卖、卖出回录、候选排序截断)与用户的核心交易哲学,本版全部并入。
>
> 定位不变:**辅助"我"做决策的系统——约束纪律、放大信息、解释概念;扳机永远由我自己扣。**

## 1. 系统形态与技术栈(锁定)

| 维度 | 决定 |
|---|---|
| 决策模式 | 系统辅助、人拍板,不全自动 |
| 客户端 | 原生 **SwiftUI iOS App**,开发者账号直装自有设备(不上架) |
| 推送 | ECS **直连 APNs** → App 系统通知(Bark 仅临时验证/备份) |

| 运行时模型 | **DeepSeek**(前置词 + skills 武装成 A 股短线专家) |
| 下单/持仓 | **不接券商 API**,手动录入(快捷指令) |

## 2. 总体架构

三块:①苹果设备　②阿里云 ECS（系统大脑）　③外部服务。

```
[苹果设备]                  [阿里云 ECS · 系统大脑]               [外部服务]
 系统通知(APNs接收) <─硬线推送─ 监控守护进程(每1min·只判硬线) ──拉价── 新浪/腾讯(实时价·免费)
 原生App(3卡片+候选列表+对话) <─看板/对话─> 后端API(过滤·排序·编排) <──> DeepSeek(深判·中间地带·复盘)
 快捷指令(开仓/清仓 双向回录) ─录入─────────> 后端API ───────────── Tushare付费(历史·资金·板块)
                                       SQLite(持仓·流水·复盘·记忆)
                                       周复盘任务(独立上下文)
                               [券商到价提醒:独立兜底,不接入系统]
```

- **闭环路径**:周复盘任务 → 写入 SQLite → 后端读出 → 注入下周交易上下文。
- **券商到价提醒**:刻意独立于系统,自制监控万一挂了仍兜底。

## 3. 脊椎:报警服务(机械层)

每 1 分钟拉在持仓票实时价,命中硬线即推送,绝不做软判断。

| 触发(硬线) | 动作 |
|---|---|
| 单票 -5% | 止损提醒(极强趋势放宽 -10% 封顶) |
| 单票 +15% | 止盈提醒(必走) |
| **持仓满 3 交易日 → 第 4 日** | **清仓提醒(无条件)** |

配套:免费实时价(多源降级兜底)；**每日收盘后"心跳"推送**；**券商到价提醒人工兜底**；**硬线主动推、中间地带状态我主动拉**。

---

## 4. 持仓录入与状态回录

- **入口**:苹果快捷指令 / Siri;App 内查看管理。
- **两种动作(只有这两种)**:
  - **开仓(全仓买入)**:录 `代码 / 买入价 / 数量 / 止损线 / 进场理由(一句话)`;系统**自动快照当时形态+资金状态**(供 Reviewer 评进场质量)。
  - **清仓(全仓卖出)**:录 `卖出价 / 时间`;系统据此停止监控该票、结算流水。**没有减仓/做 T。**
- **为什么要回录**:A 股无券商 API,系统不知道我何时真的卖了;不回录则监控会空盯一支已清的票、流水也不准。**这是状态闭环的必需一环。**

## 5. 选股系统(三段式:过滤 → 排序截断 → on-demand 深判)

> 关键自洽点:大盘好时可能上百只合格。光"过滤"不够,必须有"**排序 + 截断**"把候选收敛到我挑得过来的量;而"深判"只对我挑中的那只做。

### 5.1 粗筛(过滤,宽松,只负责"别漏")
- 形态近似:放量突破 / 创 N 日新高 / 站上关键均线(宽区间,不卡死数)。
- 资金宽条件:近几日主力净流入为正(**用截至昨日的 EOD 数据**;盘中拿不到当日资金)。
- **硬排除**:
  - 黑名单:**300、688、白酒、ST**。
  - **高位排除**:前期累计涨幅 **≥100% → 不做**;**≥50% → 警告(降级)**。**只做低位平台。**
  - 资金硬规则:**当日小单净流入特别多 → 排除**(散户接盘/疑似出货)。⚠️ 当日资金为 EOD 数据,此条**收盘后/次日校验**,非盘中。

### 5.2 排序 + 截断(机械层)
- **排序方式 = 加权混合**:四因子都算分、**突破放量强度权重最大**,其余依次 资金面(主力净流入)> 换手率/活跃度 > 低位程度。(不卡死数;具体权重首版经验值、复盘迭代。)
- **截断 = 5 × 空仓位数(上限)**:
  - 全空仓(3 位)→ 前 15;空 1 位 → 前 5;**满仓 → 0(闭嘴,注意力放在已持仓的票上)**。
  - **下限为 0**:当天零合格候选 → **不操作**(唯一的"歇"条件)。
- 说明:排序分**不是死阈值、不卡生死**(生死交给 5.1 的宽条件),只决定"先看谁"。

### 5.3 呈现 + on-demand 深判 + 决策
- App 呈现一个**硬排序候选列表 + 轻量数据**(现价 / 放量 / 主力净流入 / 换手)。
- **我自己挑**:看中哪只,问 LLM "分析一下这个" → DeepSeek 才做**深度分析**(形态→资金→消息排雷→拿不拿建议,B 剂量)。
- 决策:满意 → **全仓买入并录入**;不满意 → 问下一只;直到挑定或本轮作罢。
- ⇒ **DeepSeek 不批量分析整张列表,只深析我挑中的那只**(省调用、贴合"挑选与扳机都在我")。

### 5.4 选股哲学(锁定)
- **任何时候尽量上仓位,炒个股不炒大盘**:不设大盘全局开关,不因大盘缩量/下跌就停手——总有逆势涨的票。
- **每天都可操作**;**唯一不操作 = 选股逻辑当天一只都筛不出来**。
- 风控不靠"择时歇手",而靠**进场质量(必过滤/排序)+ 离场铁律**两道闸。

## 6. 三维度细则(DeepSeek 深判时的"专业弹药")

### 6.1 形态面(主轴)
- 平台突破 / 底部放量启动;**剔除左侧抄底**。
- **进场时机:A/C/D 有机结合,按看盘时机自适应**(A 撞上盘中突破 / C 尾盘放量站稳 / D 昨日没进次日仍强);**B 等回踩排除**(强势股摸不到)。
- 平台定义 / 突破有效性:**不设死阈值**,LLM 具体分析(宽区间辅助)。
- 止损位:以 **-5%** 为准;破平台但整体仅亏 2–3% → 中间地带由 LLM 判。

### 6.2 资金面(确认器)
- **只看 主力净流入 + 换手率**(不看北向、龙虎榜)。
- **连续几日净流入**(持续性优先),**当日不能大幅净流出**。
- 顺序:**先 K 线 → 再资金确认**。
- **数据时序**:moneyflow 为 EOD,盘中只用到昨日为止;**当日资金(小单爆量/净流出)收盘后或次日校验**;不爬盘中资金流(避免爬虫不稳)。

### 6.3 消息面(排雷,非入口)
- 资金 + 技术是主轴;消息/板块只做**最后排雷**。
- 题材热度:**板块涨幅 + 板块资金流入**(结构化,不碰新闻 NLP)。
- **一日游判断**:LLM + **读东财股吧 + 雪球舆情**感知情绪(best-effort、低频、仅作参考,仅对我挑中的候选)。
- 个股消息:**不作买入理由,只排雷**(如被监管警告 → 不进)。
- **"泡沫明显" = LLM 看「短期暴涨/乖离过大」+「情绪过热(舆情狂欢)」**,**不看估值 PE**(不误杀高估值题材股)。

## 7. LLM 决策层(DeepSeek)

- **前置词 + skills**:把第 6 节方法论写成系统提示 + 技能模块。
- **中间地带(-5%~+15%)= B 剂量,二元建议**:给信息 + **明确建议"拿 or 清"**(无减仓选项)+ 写清理由;扳机我扣。
  - 最看重:**量能是否萎缩 + 主力资金还在不在**(最优先,非唯一);兼看是否仍在买入逻辑、大盘环境、距两条线远近。
- **利润回撤保护**:不设自动规则;冲高回落时**我主动问**,LLM **看分时线趋势**帮我分析。
- **反情绪教练(启用)**:发现我想破纪律硬扛("总觉得会反弹")时,**主动调出复盘历史 + 铁律怼我**;与闭环联动。

---

## 8. 离场规则

- 止损:**-5% 必走**(极强趋势 -10% 封顶)。
- 止盈:**+15% 必走**。
- **时间止损:满 3 交易日,第 4 日无条件清仓。**
- 中间地带(-5%~+15%):LLM 辅助,二元"拿 or 清"。
- 所有离场 = **全仓清出**;因 T+1,动作集中在次日。

---

## 9. Reviewer 周复盘模块

- **隔离**:独立 Agent(DeepSeek,独立上下文),**只吃客观流水 + 铁律,不读盘中聊天**;**仅周末触发**(控制时间投入,不做盘中/当日小结);周末前提醒。
- **双维度评判**:
  - **纪律(过程)**:守纪律=肯定/不标红,破纪律=标红。
  - **盈亏(结果)**:赚钱总结经验,亏钱想教训。
- **打分**(量化分数 + 标红标签):
  - 维度:①进场合规 ②离场守纪律 ③中间地带有无情绪化硬扛 ④是否破铁律 ⑤**结果(盈亏/落袋质量)计入**。
  - 标尺:**100 = 守纪律 + 吃满 +15% 落袋 + 走得坚决**;**-5% 止损即便纪律满分也仅 80–90**(进场/选股质量可提升);具体分数 LLM 定。
- **标红清单**:该走没走 / 超 3 日没清 / 追了高位(进 ≥50% 警告区)/ 进场不符合形态资金 / 没记理由凭感觉乱进。
- **肯定清单**:亏钱但严格止损 / 守住第 4 日清仓 / 该走就走哪怕错过后续反弹。
- **产出**:每周一份**小报告** = 每笔点评 + **纪律执行率(带趋势)** + 盈亏经验/教训 + 一条"下周注意"。
- **闭环(A)**:结论自动写入 SQLite,注入下周交易上下文。

## 10. 数据层

- **实时价(脊椎)**:免费新浪/腾讯,多源降级兜底。
- **分时/分钟线**:免费,按需(利润回撤分析时拉)。
- **Tushare 付费 = 2000 积分会员(约 200 元/年)**,已确认覆盖所需标准接口:`moneyflow`(主力/小单净额)、`daily_basic`(换手率/涨跌幅/PE-PB)、`daily`(日线·形态)、`trade_cal`(交易日历)。**无需任何"独立权限"加购**(实时价、分钟线均走免费源;新闻/公告类不接)。板块级资金流向(`moneyflow_ind_*`)卡 5000 积分,暂不升档(消息面排雷用免费板块涨幅替代)。
- **舆情**:股吧/雪球/博主,仅对我挑中的候选少量抓取(best-effort)。

## 11. 存储(SQLite 表草案)

> 持仓为"全有全无",无部分仓位字段。

- `positions`:当前持仓(代码、买入价、数量、止损线、进场理由、进场形态/资金快照、买入日、持仓交易日计数)。**最多 3 行。**
- `trades`:历史流水(开仓价/清仓价/时间、是否守各条线、盈亏、是否破纪律)。**每笔一买一卖闭合。**
- `reviews`:每周复盘结论(评分、标红项、纪律执行率、经验/教训、下周注意)。
- `memory`:注入交易上下文的闭环结论 / 长期记忆。

## 13. 已决策的"砍掉/不做"项

- **全自动回测:砍。** 决策是"系统辅助、人拍板"。
- **不接券商下单/持仓 API**(A 股无开放 API)。
- **无加仓/减仓/做 T**(全买全卖)。
- **无大盘全局开关**(尽量上仓位)。
- 历史重放沙盒推到 V2(陪练非裁判)。
- 若将来做任何回测,避三坑:**涨停买入假设、未来函数、交易成本**。

## 15. 主要风险

- **自制监控悄悄挂** → 每日心跳 + 券商到价提醒双兜底。
- **"尽量上仓位"偏积极** → 风控全压在"进场质量(过滤+排序)+ 离场铁律"两道闸上,而非择时歇手;若进场闸不够严,可能交易过频。复盘的纪律执行率/盈亏双维度是纠偏手段。
- **免费实时接口不稳/限频** → 多源备份。
- **DeepSeek 调用**:国内 ECS 直连无障碍;若改用 Claude 需自建中转。
- **V2 沙盒纪律迁移风险**:知道是历史 + 无真金白银压力,纪律未必迁移;选段有幸存者偏差。
